Uzak Sunucu Projesinde Local Değişikliği Yayınlamak

August 1, 2021

Projemizde yapılan değişikliklerin staging ve commit işlemleri yaptıktan sonra tüm bu değişiklikleri projemizin stabil versiyonu olan master branch ile merge etmemiz yani entegre işlemi gerekir.

Commit işlemi ile dosyalarınızda yapılan değişiklikler kalıcı olarak repository'de kayıt altına alınınr. Ancak günlük çalışmamızda üzerinde çalışma esnasında uzak sunucu repository'sinde bulunan değişikleri yerel diske çekme işlemi yapmak gerekebilir. Bu durumda yapılan değişiklikleri geçiçi olarak kaydedilmesi gerekir.

git stash komutu ile üzerinde çalıştığınız ancak henüz commit etmediğiniz değişikliklerin geçici olarak Git tarafından kayıt altına alınmasını ve aktif branch'inizin herhangi bir değişikliğin olmadığı temiz bir duruma getirilmesini sağlarsınız.

Lokaldeki bir branch üzerindeki yaptığımız değişikleri uzak sunucuya atmadan önce yapılması gerekenler için bu komut kolaylık sağlamaktadır. Bunun için ;

Git komutlarını adım adım inceleyelim.

  • git stash Lokal branchdeki yapılan değişiklikleri geçiçi olarak kaydetme.
  • git checkout master Master branchine geçiş yapma
  • git pull Uzak repositorydeki master branchindeki yapılan değişiklikleri lokale çekme
  • git checkout branch Lokalde çalışılan branche geçiş yapma.
  • git merge master Master branchindeki çekilen değişiklikleri lokaldeki branche çekme işlemi
  • git stash pop Lokal branchindeki geçiçi olarak kaydedilen değişiklikleri alma

Bu adımlarla birlikte lokaldeki projemize uzak sunucudaki repository eşitlenmiş oldu. Daha sonra değişiklikleri tamamlayarak commit ve push işlemi ile yayınlanabilir.
